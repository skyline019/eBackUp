\chapter{阶段四：Wave 5 与 Sprint 4（\versiontag{0.9.0}--\versiontag{0.9.4}）}
\label{ch:phase-streaming}

\section{\versiontag{0.9.0}：正确性优先于分数}

\begin{pressbox}[P3 爆发]
Pipeline v4 多 worker 下 ChunkStore 全局 \texttt{index\_mu\_} 导致 heap corruption（\texttt{0xC0000005}/\texttt{0xC0000374}）与 \texttt{ForEachRecord} 重入死锁——在 L5 $\sim$146 MB/s 时若继续 chase 分数，生态将因 \textbf{不可恢复 crash} 归零。
\end{pressbox}

\textbf{数据对照}（同版）：
\begin{itemize}[nosep]
  \item L3b ratio \textbf{0.96--0.98}：曾险触 immutable 0.90 floor；
  \item L5 \textbf{142--149}：相对 v0.8 146 略波动；
  \item profile：chunk 占 pipeline \textbf{98\%}——指向拓扑而非 store。
\end{itemize}

\textbf{Stage 3 分层数据}：v0.9.0 曾写入 \texttt{ci\_floor.json} 草案 L5=\textbf{280}；v0.9.1 下调 Stage 3.1 至 L5=\textbf{105}，Stage 3.2 保留 280（见 \cref{ch:measured-data}）。

\section{阶段小结}

\versiontag{0.9.x} 四轮 sprint 合计 L5：145 $\rightarrow$ 171（+\textbf{18\%}），L7：598 $\rightarrow$ 653（+\textbf{9\%}）。单文件瓶颈仍在 cdc 82\%——Stage 3.2 的 280 MB/s 需 $\mathbf{+109}$ MB/s（\cref{ch:measured-data}），这是 Sprint 5 的量化目标而非口号。
