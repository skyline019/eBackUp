\chapter{不变量、Parity 与完整叙事闭环}\label{ch:parity}

\section{三大不变量}

\begin{invariantbox}[I-B1：Content ID]
$\text{Content ID} = \texttt{SHA256}(\text{chunk 明文})$。digest 算法可演进，但 cut 边界必须先锁定。
\end{invariantbox}

\begin{invariantbox}[I-B2：Cut-point Parity]
\texttt{FastCdcStreamFeed}、\texttt{FastCdcSlice::ChunkCuts}、\texttt{ChunkCutsUntil}、SIMD 路径对相同输入产生\textbf{相同 offsets/lengths}。
\end{invariantbox}

\begin{invariantbox}[I-B3：Manifest 耐久性]
chunk hash 列表写入 manifest 前 \texttt{Flush}/fsync；txn 中断 $\rightarrow$ \texttt{kAborted}，不提交半成品。
\end{invariantbox}

\section{Algorithm Change Policy}

\begin{enumerate}
  \item 改动 \texttt{fast\_cdc*.cc} 或 \texttt{fast\_cdc\_streaming.cc}；
  \item 本地跑 \texttt{fast\_cdc\_streaming\_test} + \texttt{streaming\_chunk\_pipeline\_test}；
  \item parity 失败 $\Rightarrow$ \textbf{回退}（Phase 3C 先例：0 MB/s 收益）；
  \item parity 通过但 L5 绝对吞吐崩溃 $\Rightarrow$ \textbf{默认关闭}（Phase A：105 MB/s）；
  \item CI immutable floor：\texttt{reuse\_pct\_min}$\geq$90\%，\texttt{pipeline\_ratio\_min}$\geq$0.90，L5 floor 105 MB/s。
\end{enumerate}

\section{30 秒 / 2 分钟答辩口播}

\begin{corebox}[30 秒版]
「分块用 FastCDC 内容定义分块，大小在 min/avg/max 间浮动；Gear mask 使期望间距 $\approx$ avg\_size。生产走 32MB StreamFeed，carry 保 gear 连续，Phase B handoff 防 drift，parity 测试立法。增量用 HCRBO/CFI 三级验证跳过 unchanged chunk。路由五路径：小文件 Inline、大文件 Streaming、多文件/增量 v4。」
\end{corebox}

\begin{corebox}[2 分钟版（CDC 漂移专集）]
「CDC 漂移是流式与整文件切点不一致，会导致 SHA256 变。Slice 是 golden，StreamFeed 是生产路径，必须 bit-identical。carry tail 解决 gear 跨 feed 断档；Phase B 在 carry 消耗完后 handoff 到 contiguous loop，禁止 virtual boundary 简化。digest\_base 保证 file-absolute hash。每次改 CDC 跑 streaming\_test 和 256MB manifest parity；失败回退，overlap 丧失的路径默认关闭。」
\end{corebox}
