\section{表初始化：\texttt{InitGearTableForConfig}}

v6 维护两张 256 项表，滑窗更新公式与 FastCDC 相同，仅表内容不同：

\begin{align}
  h_{\mathrm{gear}} &\leftarrow (h_{\mathrm{gear}} \ll 1) + \mathrm{gear}[b] - \mathrm{gear}[b-w] \\
  h_{\mathrm{norm}} &\leftarrow (h_{\mathrm{norm}} \ll 1) + \mathrm{norm}[b] - \mathrm{norm}[b-w]
\end{align}

\begin{lstlisting}[caption={v6 norm 表种子（\filepath{gt\_cdc.cc}）},label={lst:init-norm-table}]
if (cfg->kernel == GtCdcKernel::kTwoFGear) {
  const uint32_t norm_seed =
      cfg->table_seed != 0 ? (cfg->table_seed ^ 0x9E3779B9u) : 0u;
  if (norm_seed != 0) {
    InitKeyedGearTable(cfg->norm_table, norm_seed);
  } else {
    InitGearTable(cfg->norm_table);
  }
}
\end{lstlisting}

\section{与 v5 seed-mix 的对比}

v5 在\textbf{单} $h$ 上做 $\texttt{MixSeedHash}(h, seed, pos)$，且 $M$ 随 chunk 内距离变化。v6 把「seed 隔离」前移到 \texttt{norm\_table} 构造，扫描循环内\textbf{无} \texttt{MixSeedHash}、\textbf{无} \texttt{SelectPhaseMask}，probe 体与 FastCDC 同量级。

\section{窗口预热}

在 \texttt{scan\_start = pos + min\_size} 处，两路 hash 均通过 \texttt{InitWindowHash(data, scan\_start, w, table)} 预热；Streaming resume 时跳过预热，直接注入 \texttt{tf\_gear\_h}/\texttt{tf\_norm\_h}（\cref{ch:streaming-resume}）。
