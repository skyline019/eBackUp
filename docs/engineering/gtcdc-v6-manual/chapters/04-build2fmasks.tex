\section{算法}

\texttt{Build2FMasks(avg\_size, nc\_level, mask\_lo, mask\_hi, norm\_shift)} 是 v6 的\textbf{核心参数化}函数，源码位于 \filepath{gt\_cdc\_gear\_scan.cc}。

\begin{lstlisting}[caption={Build2FMasks 完整实现},label={lst:build2fmasks}]
void Build2FMasks(uint32_t avg_size, uint8_t nc_level,
                  uint32_t* mask_lo, uint32_t* mask_hi,
                  uint32_t* norm_shift) {
  uint32_t total_bits = 0;
  uint32_t v = avg_size;
  while (v > 1) { v >>= 1; ++total_bits; }
  if (total_bits == 0) total_bits = 1;
  if (total_bits > 31) total_bits = 31;

  uint32_t hi_bits = total_bits / 2;
  if (nc_level > 0) {
    const uint32_t nc_cap =
        total_bits / (static_cast<uint32_t>(nc_level) + 1u);
    if (nc_cap > 0 && nc_cap < hi_bits) hi_bits = nc_cap;
  }
  if (hi_bits == 0) hi_bits = 1;
  uint32_t lo_bits = total_bits - hi_bits;
  if (lo_bits == 0) { lo_bits = 1; if (hi_bits > 1) --hi_bits; }

  *mask_lo = (1u << lo_bits) - 1u;
  *mask_hi = (1u << hi_bits) - 1u;
  *norm_shift = lo_bits;
}
\end{lstlisting}

\section{早期缺陷与修复}

\begin{v6box}[v6 关键 bug：满宽双 AND]
若错误地对 $h_{\mathrm{gear}}$ 与 $h_{\mathrm{norm}}$ 各用完整 $\log_2(\texttt{avg})$ bit mask 再 AND，则
$P(\text{hit}) \approx 2^{-2B}$，切点几乎永不出现 $\Rightarrow$ 大量 max-cut、块长 $\approx$ \texttt{max\_size}、probe 统计异常。\textbf{修复}：将 $B$ bit \textbf{拆分}到 $M_{lo}$ 与 $M_{hi}$（\cref{fig:bit-domains}）。
\end{v6box}

实测（256\,MB 随机，Large profile）：修复后 2f\_gear 约 197 chunks（mean $\approx$1.36\,MB），与目标 avg 同阶；修复前 hit$\approx 0$ 时 chunks 数极低且 mean$\rightarrow$max。
