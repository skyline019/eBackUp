\section{Wave B/E：Manifest v5 与 Windows 元数据}

\subsection{Manifest v5 触发条件}

\texttt{WriteManifestAuto} 在 \texttt{DocumentUsesV5(doc)} 时写入 \texttt{EBMANIFEST5}（见 \filepath{manifest.cc}）。任一文件含 Win 扩展字段即触发 v5。

\subsection{扫描 enrichment}

\filepath{scan\_entry.cc} 在 \texttt{\_WIN32} 下调用 \texttt{EnrichScanEntriesWinMeta}：
\begin{itemize}[nosep]
  \item 常规文件采集 ACL、inode、reparse；
  \item \texttt{AppendAlternateStreams}：\texttt{FindFirstStreamW} 枚举 ADS，\texttt{relative\_path} 追加 \texttt{:stream}。
\end{itemize}

\subsection{恢复}

\filepath{restore\_engine.cc}：硬链 dedup（\texttt{CreateHardLinkW}）；联接点 \texttt{RecreateReparsePoint}；ADS 写入 \texttt{path:stream}。

\section{Wave I：Hybrid CDC}

默认路径（\versiontag{0.10+}）：\texttt{FastCdcSlice} 切分 + stream replay 混合。环境变量：
\begin{itemize}[nosep]
  \item \texttt{EBBACKUP\_CDC\_HYBRID=0} — 禁用混合，回退纯 stream；
  \item \texttt{EBBACKUP\_FORCE\_STREAM\_CDC=1} — 强制 stream-feed。
\end{itemize}

\texttt{PipelinePhaseStats}：\texttt{hybrid\_cuts\_ns}、\texttt{hybrid\_replay\_ns} 与 \texttt{stream\_cdc\_ns} 互斥计数——测试断言须 OR 组合。

\section{Wave M：三路就地恢复}

Preview JSON 分类：\texttt{added}、\texttt{removed}、\texttt{modified}、\texttt{conflict}、\texttt{both\_changed}。

C API：\texttt{eb\_backup\_preview\_in\_place\_json} / \texttt{eb\_backup\_apply\_in\_place\_json} 支持 \texttt{base\_txn\_id}、\texttt{use\_three\_way}、\texttt{dry\_run}。

\section{Wave P/Q：插件接口}

\section{Wave R：智能排除}

\filepath{exclude\_suggestions.cc} catalog 含 \texttt{.git}、\texttt{node\_modules}、\texttt{.next}、\texttt{.turbo}、\texttt{Thumbs.db}、\texttt{*.log} 等。

C API \texttt{eb\_backup\_suggest\_exclude\_filters\_json} 参数 \texttt{include\_ide\_dirs} 控制 \texttt{.vscode}/\texttt{.idea} 建议。

\section{Wave T：备份窗口}

\filepath{job/backup\_window.cc}：
\begin{itemize}[nosep]
  \item \texttt{IsWithinBackupWindow(start, end, now)} — 支持跨午夜；
  \item \texttt{ShouldDowngradeDurability(end, grace, now)}；
  \item \texttt{WindowEndUnix(...)} — 报告字段。
\end{itemize}

\texttt{RunBackup} 中途 \texttt{MaybeAdaptBackupWindow}；超时 \texttt{TruncatePendingFilesAt}。
